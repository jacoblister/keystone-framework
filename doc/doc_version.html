Installing Developers Reference Version History

Version History

Keystone uses a version control numbering scheme simular to that used to maintain the Linux Kernel. That is, the version number has four parts

Major.Minor.Revision-SubRevision

Major Version numbers indicate very significant milestones in the development

  • Odd Minor version numbers indicate a development version. Once stable, this will version will become renumbered as the following Major (even) number.
  • Even Minor version numbers indicate a release version. Changes in release version are intended only for bugfixes, and not for new development
Revision and SubRevision Version numbers indicate development and bug fixes under a Minor Version
